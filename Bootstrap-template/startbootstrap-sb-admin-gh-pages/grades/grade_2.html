OOP Basics

Introduction to OOP

Object-Oriented Programming (OOP) organizes code into objects that represent real-world entities. Each object has two aspects: state (data) and behavior (methods).

Goals of OOP

  • Reusability: Create reusable components.
  • Single Responsibility: Each object is responsible for one task.
  • Encapsulation: Protect data and logic inside objects.
  • Extensibility: Add new features without modifying existing code.

General Class Example

Here's a general example of a class that represents a bank account with methods to deposit, withdraw, and check the balance.

Example in C#:

public class BankAccount {
    private string _owner;
    private decimal _balance;

    // Constructor
    public BankAccount(string owner, decimal initialBalance) {
        _owner = owner;
        _balance = initialBalance;
    }

    // Deposit money
    public void Deposit(decimal amount) {
        _balance += amount;
    }

    // Withdraw money
    public void Withdraw(decimal amount) {
        _balance -= amount;
    }

    // Display account info
    public void DisplayInfo() {
        Console.WriteLine($"Owner: {_owner}, Balance: {_balance:C}");
    }
}

// Usage
BankAccount account = new BankAccount("Alice", 1000);
account.Deposit(200);
account.Withdraw(150);
account.DisplayInfo(); // Output: Owner: Alice, Balance: $1,050.00

Constructors

A constructor is a special method in a class that initializes objects of that class. It has the same name as the class and does not have a return type. Constructors are used to set the initial state of an object.

Example in C#:

public class Human {
    private string _name;
    private int _age;

    // Constructor
    public Human(string name, int age) {
        _name = name;
        _age = age;
    }

    public void DisplayInfo() {
        Console.WriteLine($"Name: {_name}, Age: {_age}");
    }
}

// Usage
Human person = new Human("Alice", 30);
person.DisplayInfo(); // Output: Name: Alice, Age: 30

Data Structures

Dynamic Data Structures

Dynamic data structures adjust memory allocation at runtime, offering flexibilit over fixed-size arrays. Examples include:

  • List: A resizable array.
  • Stack: Last In First Out (LIFO).
  • Queue: First In First Out (FIFO).
  • Set: Stores unique elements.
  • Map: Stores key-value pairs.

Self-Implemented List with Nodes

A self-implemented list can use nodes to store data. Each node links to the next node, forming a chain.

public class Node {
    public int Data { get; set; }
    public Node? Next { get; set; }

    public Node(int data) {
        Data = data;
        Next = null;
    }
}

public class LinkedList {
    private Node? _head;

    public void Add(int data) {
        if (_head == null) {
            _head = new Node(data);
        } else {
            Node current = _head;
            while (current.Next != null) {
                current = current.Next;
            }
            current.Next = new Node(data);
        }
    }

    public void PrintAll() {
        Node? current = _head;
        while (current != null) {
            Console.WriteLine(current.Data);
            current = current.Next;
        }
    }
}

// Usage
LinkedList list = new LinkedList();
list.Add(1);
list.Add(2);
list.Add(3);
list.PrintAll();

Dictionary (Manual Implementation)

A dictionary can be implemented manually using a class that stores key-value pairs. The class includes methods for adding, retrieving, and updating values.

Manual Implementation in C#:

public class ManualDictionary {
    private List _keys = new List();
    private List _values = new List();

    // Add a key-value pair
    public void Add(string key, int value) {
        _keys.Add(key);
        _values.Add(value);
    }

    // Update a value
    public void Update(string key, int newValue) {
        int index = _keys.IndexOf(key);
        _values[index] = newValue;
    }

    // Retrieve a value
    public int Get(string key) {
        int index = _keys.IndexOf(key);
        return _values[index];
    }

    // Display all key-value pairs
    public void DisplayAll() {
        for (int i = 0; i < _keys.Count; i++) {
            Console.WriteLine($"Key: {_keys[i]}, Value: {_values[i]}");
        }
    }
}

// Usage
ManualDictionary dictionary = new ManualDictionary();
dictionary.Add("Alice", 30);
dictionary.Add("Bob", 25);
dictionary.Update("Alice", 31);
dictionary.DisplayAll(); // Output: Key: Alice, Value: 31 | Key: Bob, Value: 25

Generics

Introduction to Generics

Generics allow you to define classes and methods with a placeholder for data types. This reduces code duplication and improves type safety.

Using Generics with Constraints

Generics can include constraints to restrict the type of data they accept. For example, you can specify that a generic type must implement a certain interface or inherit from a base class.

Example in C#:

public class GenericCalculator where T : struct {
    public T Add(T a, T b) {
        return (dynamic)a + (dynamic)b;
    }
}

// Usage
var calculator = new GenericCalculator();
Console.WriteLine(calculator.Add(5, 10)); // Output: 15

Benefits of Generics

  • Reusability with type parameters.
  • Type safety at compile time.
  • Performance improvements with value types.

Example: Generic List

public class CustomList {
    private T[] _items;
    private int _count;

    public CustomList(int capacity) {
        _items = new T[capacity];
        _count = 0;
    }

    public void Add(T item) {
        _items[_count++] = item;
    }

    public T Get(int index) {
        return _items[index];
    }
}

// Usage
CustomList myList = new CustomList(5);
myList.Add("Hello");
myList.Add("World");
Console.WriteLine(myList.Get(1)); // Output: World
